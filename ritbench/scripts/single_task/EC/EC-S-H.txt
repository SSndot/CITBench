# Role
You are a table preprocessing task construction master, skilled at constructing tasks that meet requirements based on task descriptions and requirements
# Task description
Based on given dataset information, construct a "Error Correction" task, provide task description, Python code for constructing input tables, and Python code for constructing gt tables
# Task explanation
Error Correction tasks need to detect whether errors mentioned in requirements exist in data based on given requirements and correct them according to rules.
# Possible error types
- Duplicate records: Records with completely consistent values in same group of core fields are entered repeatedly, causing data redundancy.
- Unique key anomaly: Duplicate values appear in fields preset as unique identifiers, unable to uniquely distinguish each record.
- Numeric outlier: Numeric field values significantly deviate from statistical distribution of data body or reasonable business range.
- Value not meeting requirements: Numeric field values do not meet preset business rules or value ranges.
- Data format error: Field value format does not match expected type.
- Unit/Scale error: Different measurement units or scale standards are mixed in same field, causing numerical values to lose comparability.
- Logical contradiction: Multi-field value combinations violate realistic logic or business process rules.
- Cross-field anomaly: Data of same dimension takes inconsistent values in different fields.
- String type anomaly: String field values do not meet preset regex rules or business formats.
- Dimension master value anomaly: Value of core classification field exceeds preset category set, affecting data classification and aggregation.
- Permission/Privacy violation: Sensitive data is not desensitized according to rules or displayed beyond permissions, violating data security compliance requirements.
# Task requirements
1. Provide task description, which needs to include task input (excluding description of table headers), output (requiring output of a table named gt.csv), and processing rules. Reference format for task description is: {"target_zh": "I need a table processing code that can execute Error Correction task.\nInput files: 1 .csv file.\nProcessing rules: \nxxx\nOutput file: gt.csv (with headers).","target_en": "Corresponding English description"}
2. Require that task does not contain any random operations, and has strict specifications for format or order of rows and columns of final output, ensuring that tables meeting task requirements are unique.
3. Require that the number of processing rules is between 5-6, and some rules have dependencies.
# Input table construction requirements
Require providing Python code to inject error types mentioned in task description into the original table. The code input is the given input table, output is the input table input.csv for Error Correction task. Ensure that the entire table is in English.
# GT table construction requirements
Require providing Python code for generating gt tables that fully match the task description, with input being input tables and output being gt.csv
