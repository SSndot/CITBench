# Task construction requirements
1. Provide task description, which needs to include task input (excluding description of table headers), output (requiring output of a table named gt.csv), and processing rules. Input must not explain of source of tables, and task description must not explain specific task types. The way to refer to tables in task description is "input1", "input2", etc., and other ways of referring to tables are prohibited. "result of step x" is prohibited in processing rules, can be replaced with "result of rule x". Reference format for task description is: {{"target_zh": "I need a table processing code that can execute tasks containing multiple table operations.\nInput files: x .csv files.\nProcessing rules: \nxxx\nOutput file: gt.csv (with headers).", "target_en": "Corresponding English description"}}
2. Require that task does not contain any random operations, and has strict specifications for format or order of rows and columns of final output, ensuring that the table meeting task requirements is unique.
3. Require that task includes all given task types, and that the same task type can have multiple subtasks with different rules and different processing objects, totaling 8-12 subtasks.
4. One subtask corresponds to one processing rule, and processing rules must be strictly arranged in the order of subtask execution.
5. Processing rules need to be set based on multiple tables simultaneously.
6. Need to ensure that numeric types are suitable for related operations.
7. Require that multiple subtasks have complex dependency relationships.
8. The final output can only be 1 gt table, which needs to be named "gt.csv".
